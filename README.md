## 搬瓦工多端口是什么意思？先说清楚

搬瓦工的 VPS 本身没有限制端口数量，你开多少个端口监听都行（合理范围内）。所谓**多端口教程**，核心就是两件事：

1. 在代理软件（V2Ray / SSR）的配置文件里，新增一个 inbound 监听端口
2. 在系统防火墙（iptables 或 firewalld）把这个端口放行

两步走完，朋友用新端口+新密码连进来，和你互不干扰。流量独立计算，断开不影响对方。

## V2Ray 多端口教程：两种方案，按场景选

V2Ray 的多用户有两个思路：**同端口不同 UUID**，或者**一人一端口**。

同端口方案：所有人走同一个端口，靠不同的 UUID 区分身份。管理起来省事，就一个防火墙规则，缺点是某个 UUID 出问题整个端口都受影响。

多端口方案：每个用户一个独立端口和密码，完全隔离。需要多开几条防火墙规则，但互不干扰，流量也好分开统计。

我自己偏向多端口。分出去的端口随时可以关掉，不用改别人的配置。

### 方案一：V2Ray 同端口不同 UUID

SSH 连进搬瓦工，打开配置文件：

bash
nano /etc/v2ray/config.json


找到 `"clients"` 这个数组，在里面追加一个新对象就行：

json
"clients": [
  {
    "id": "原来的UUID",
    "level": 1,
    "alterId": 64
  },
  {
    "id": "新生成的UUID",
    "level": 1,
    "alterId": 64
  }
]


新的 UUID 可以在服务器上用这条命令生成：

bash
cat /proc/sys/kernel/random/uuid


复制出来填进去。改完保存，重启 V2Ray：

bash
systemctl restart v2ray


这个方案不需要改防火墙，端口没变。朋友只需要一个新的 UUID，其他配置和你的一样。

### 方案二：V2Ray 多端口独立用户

还是编辑 `/etc/v2ray/config.json`，在 `"inbounds"` 数组里追加一段新的监听配置：

json
"inbounds": [
  {
    "port": 原来的端口,
    "protocol": "vmess",
    "settings": {
      "clients": [{"id": "你原来的UUID", "level": 1, "alterId": 64}]
    }
  },
  {
    "port": 新端口号,
    "protocol": "vmess",
    "settings": {
      "clients": [{"id": "新生成的UUID", "level": 1, "alterId": 64}]
    }
  }
]


新端口号随便选，建议 10000-65535 之间，避开常见端口。改完重启 V2Ray。

然后必须放行这个新端口——这步很多人忘掉，连不上才反应过来。

## 搬瓦工防火墙端口放行：这步不做就是白配

这是**搬瓦工多端口教程**里最容易漏掉的一步。配置文件改完、V2Ray 也重启了，朋友就是连不上——大概率是防火墙没放行。

搬瓦工的 VPS 系统防火墙要单独操作，具体命令看你装的是哪个系统：

**CentOS 7 / Debian / Ubuntu（用 firewalld）：**

bash
firewall-cmd --zone=public --add-port=新端口号/tcp --permanent
firewall-cmd --reload


**CentOS 6 / 旧系统（用 iptables）：**

bash
iptables -A INPUT -p tcp --dport 新端口号 -j ACCEPT
service iptables save


放行之后测一下是否生效：

bash
firewall-cmd --list-ports   # firewalld 用这个
iptables -L INPUT -n        # iptables 用这个


能看到新端口在列表里就行。

说句题外话：搬瓦工的 KiwiVM 面板本身没有安全组这个概念，不需要在面板里额外点什么"开放端口"——只要系统防火墙放行就够了，和阿里云、腾讯云那种要在控制台单独配安全组不一样。

## SSR 多端口教程：改 config.json，结构不同

用 ShadowsocksR 的流程类似，但配置文件格式不一样。SSR 的多用户本质上就是多端口——每个端口对应一个密码，没有 UUID 这个概念。

SSR 配置文件通常在 `/etc/shadowsocks-r/config.json`，找到里面的端口和密码字段，改成这种格式：

json
{
  "server": "0.0.0.0",
  "port_password": {
    "8381": "密码1",
    "8382": "密码2",
    "8383": "密码3"
  },
  "timeout": 300,
  "method": "aes-256-cfb"
}


原来单端口的两行 `server_port` 和 `password` 要删掉，替换成 `port_password` 这个对象。每加一个用户就加一行。

改完重启 SSR：

bash
/etc/init.d/shadowsocks-r restart


然后用同样的方式放行每个新端口。这一步和 V2Ray 的方法完全一样，firewalld 或者 iptables 挑一个用。

## 常见问题排查

**配完之后朋友就是连不上，怎么查？**

按这个顺序查：

1. 确认 V2Ray 或 SSR 进程在跑：`systemctl status v2ray` 或 `ps aux | grep shadowsocks`
2. 查一下 V2Ray 的错误日志：`cat /var/log/v2ray/error.log`，JSON 格式错误在这里会有提示
3. 确认防火墙已放行：`firewall-cmd --list-ports`
4. 在另一台机器上 telnet 测试端口是否通：`telnet 服务器IP 新端口号`

**配置文件改完后 V2Ray 启动失败怎么办？**

JSON 格式挑剔，少一个逗号或者括号都会报错。最快的办法是把配置文件复制出来，在 JSON 格式校验工具里贴一下，哪里有语法问题立刻能看到。改完再上传回服务器。

**多端口会影响原来的速度吗？**

不会。每个端口独立监听，互不干扰。带宽按整台 VPS 共享，如果同时有人猛跑流量，你自己那条会稍微慢一点，但这是物理层面的，和端口数量无关。

**搬瓦工换了 IP 之后原来的多端口配置还能用吗？**

能用。IP 变了，朋友那边只需要改服务器 IP，端口和密码/UUID 不用动。KiwiVM 面板里有换 IP 的功能，换完通知朋友改一下客户端就行。

## FAQ：搬瓦工多端口常见问答

**Q：搬瓦工支持开多少个端口？**  
没有明确的数量上限。实际上受限于系统资源，正常家用分享的话开十几个端口完全没问题。

**Q：搬瓦工的 KiwiVM 面板需要单独开放端口吗？**  
不需要。KiwiVM 没有安全组功能，端口管理完全在系统内部（firewalld / iptables），操作完系统防火墙就行了。

**Q：V2Ray 多端口方案和同端口多 UUID 哪个更稳？**  
多端口方案相对更稳——单个端口出问题不影响其他用户，也方便随时踢掉某个用户（直接关那个端口的防火墙规则）。

**Q：SSR 和 V2Ray 能同时跑在同一台搬瓦工上吗？**  
可以，只要端口不冲突。两个服务各自监听不同端口，互不干扰。
